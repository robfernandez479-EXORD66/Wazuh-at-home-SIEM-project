# Configurations

This folder contains configuration files used in the Wazuh-at-home SIEM environment.  
These configurations support detection engineering, log collection, and endpoint visibility across Windows and Linux systems.

---

## 1. Purpose of This Folder

The goal of this directory is to store all configuration artifacts that define how Wazuh collects, parses, and analyzes security events.  
This includes:

- Wazuh Manager configuration files  
- Wazuh Agent configuration files  
- Sysmon configuration (Windows)  
- File Integrity Monitoring (FIM) settings  
- Local rules (`local_rules.xml`)  
- Any custom decoders or parsers  

---

## 2. Typical Files Stored Here

| File | Description |
|------|-------------|
| `local_rules.xml` | Custom Wazuh rules used for detection engineering |
| `ossec.conf` | Main Wazuh Manager configuration |
| `agent.conf` | Agent-side configuration for log collection |
| `sysmon-config.xml` | Sysmon configuration for Windows endpoint visibility |
| `fim.conf` | File Integrity Monitoring configuration |

---

## 3. How These Configs Support the Lab

These configuration files enable:

- Enhanced visibility into Windows and Linux systems  
- Custom detection logic aligned with MITRE ATT&CK  
- Improved event collection (PowerShell, Sysmon, authentication logs)  
- Consistent behavior across all enrolled endpoints  

---

## 4. Notes for Reviewers

If you are reviewing this portfolio:
